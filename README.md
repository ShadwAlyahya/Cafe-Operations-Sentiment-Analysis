# ☕ Cafe-Operations-Sentiment-Analysis

An end-to-end data analytics project using Python for customer sentiment analysis and Power BI to build an interactive sales performance dashboard with strategic business recommendations.

This project is a comprehensive, end-to-end data analysis initiative designed to showcase my practical skills in data processing, diagnostic analysis, database queries, and creating interactive reports.

Through this project, my goal was to push past traditional boundaries of analysis by deliberately handling complex datasets, interactive challenges, and real-world data cleaning and preparation scenarios.

As a data analyst, my primary goal was to demonstrate my passion for digging deeper, far beyond superficial numbers. I wanted to experience the full data lifecycle—starting from cleaning and structuring, all the way to advanced sentiment analysis and strategic data-driven storytelling.

### 3. Overcoming Data Engineering and Cleaning Challenges

The practical technical obstacles I successfully overcame during programming and data preparation:

* **Handling Missing and Corrupted Values:**
  During the data cleaning phase using Python, I dealt with missing values and mismatched data types in the reviews dataset. I applied automated data imputation strategies and validation checks to ensure the integrity of the dataset structure before starting any analysis.

* **Text Standardization and Noise Filtering:**
  Raw feedback contained significant noise, spelling variations, and generic words. I developed a Python script to standardize text, handle spelling inconsistencies, and programmatically remove irrelevant terms. This ensured that only high-value keywords were passed to my sentiment analysis model.

### 4. Interactive Dashboards in Power BI

Finally, I built a two-page interactive dashboard with an elegant executive design:

* **Page 1: Sales & Financial Performance**
  This page tracks total revenue ($5.44K) and profits ($3.53K), while monitoring a critical gap of (-99.5%) against an ambitious target of $700K. It illustrates monthly trends, product category sales, and performance across different branch locations (e.g., Hell's Kitchen and Astoria).
* **Page 2: Customer Feedback & Sentiment Analysis**
  This page integrates a custom word cloud displaying customers' main concerns and recurring experiences (morning, time, slow, understaffed, overpriced). It also features a matrix table linking product ratings with review counts.

### 📌 5. Strategic Recommendations

To prove that data analysis only achieves its full value when it guides actual decisions, I wrote high-level strategic recommendations based on the integration of both financial data and sentiment analysis:

* **Morning Operations and Staffing Allocation:** Optimize employee scheduling during "morning peak hours" to address recurring complaints about slowness and wait times.
* **Price Adjustments and Value Proposition:** Review the pricing of low-rated menu items, and introduce morning bundle offers (e.g., coffee + pastry) to increase the customer's perceived value and the average ticket size.
* **Replicating and Scaling Success Stories:** Analyze and scale the operating model and service quality of high-performing branches, applying them to lower-performing locations to ensure consistent revenue growth and a unified brand experience.
